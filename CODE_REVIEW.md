# `main.rs` 라인별 코드 리뷰 & 학습 노트

> **학습 팁**: 각 항목은 `main.rs`에서 등장하는 순서를 따라가며, 가능한 한 세밀한 라인 범위를 명시했습니다. 파일을 열어 두고 `코드 → 설명`을 번갈아 보면 이해가 빨라집니다.

## 1~16: 진입점과 에러 처리
- `1-4`: 표준 라이브러리 모듈을 `use`로 가져옵니다. 경로/입출력/환경 접근을 위해 필요한 기본 준비입니다.
- `6-7`: 외부 크레이트 `rpassword`, `serde_json`을 사용하여 비밀 입력과 JSON 파싱을 처리합니다.
- `9`: `type CliResult<T> = Result<T, String>`으로 에러 타입을 단순화합니다. 러스트의 제네릭 타입 별칭 예시입니다.
- `11-16`: `main` 함수. 내부 `run()`이 실패하면 에러 메시지를 표준 오류에 출력하고 종료 코드를 1로 설정합니다. 러스트 CLI에서 자주 쓰는 패턴입니다.

## 18~125: `run` 함수의 전체 흐름
- `18-20`: CLI 인자를 `Vec<String>`으로 수집하고 `parse_args`로 해석합니다.
- `22-25`: `--help`가 들어오면 도움말을 출력하고 조기에 종료합니다.
- `27-38`: 현재 작업 디렉터리를 기준으로 계약 디렉터리 경로를 확정합니다. 상대 경로를 절대 경로로 바꾸는 예시입니다.
- `35-41`: 대상 디렉터리가 없다면 즉시 에러를 반환합니다.
- `42-50`: `discover_contracts`로 계약 정보를 찾고, 결과가 비어 있으면 안내 메시지 후 종료합니다.
- `52-55`: `--list` 옵션이면 계약 목록만 출력하고 종료합니다.
- `57-61`: 사용자가 선택한 계약명을 확인합니다. 없다면 에러를 반환합니다.
- `63-69`: `--args` JSON이 있으면 파싱하고, 그렇지 않다면 필요한 경우 콘솔에서 입력을 받습니다.
- `71-77`: 입력된 생성자 인자 수를 실제 생성자 매개변수 수와 비교하여 검증합니다.
- `80`: 프라이빗 키를 옵션에서 직접 받거나 대화형으로 요청합니다.
- `82-94`: 출력 디렉터리를 준비합니다. 없으면 생성하고, 에러 메시지에 경로 정보를 포함합니다.
- `97-108`: 생성될 스크립트 파일 경로를 만들고, 이미 존재하는 경우 `--force` 없이는 덮어쓰지 않습니다.
- `110-112`: Solidity 계약과 스크립트 사이의 상대 `import` 경로를 계산합니다.
- `113-117`: `render_script`로 문자열을 만들고 파일에 기록합니다.
- `118-123`: 사용자에게 생성된 스크립트의 상대 경로를 출력합니다.

## 128~169: 데이터 구조 정의
- `128-138`: CLI 옵션을 담는 `Options` 구조체. 기본 구현으로 나머지 필드를 초기화합니다.
- `140-144`: 생성자 매개변수를 표현하는 `ConstructorParam` 구조체 (원본 문자열과 이름 분리 저장).
- `146-153`: 계약 정보 `ContractInfo`. 이름, 경로, `pragma` 등을 저장합니다.
- `155-168`: `ContractInfo::constructor_signature`는 저장해 둔 매개변수 목록을 문자열로 결합합니다.

## 171~259: `parse_args`
- `172-176`: 기본값이 채워진 `Options` 인스턴스 생성.
- `178-200`: 인자 반복자에서 옵션을 하나씩 꺼내 `match`로 분기합니다.
- `185-223`: `--contracts-dir`, `--output-dir`, `--args` 처리. `--flag=value`와 `--flag value` 두 형태 모두 지원합니다.
- `224-239`: `--private-key` 처리. 공백만 입력하면 에러를 냅니다.
- `240-255`: `--` 이후 첫 토큰 또는 첫 번째 위치 인자를 계약 선택으로 저장합니다. 두 번 이상 입력하면 에러입니다.

## 261~278: `print_help`
- 프로그램 이름과 사용법, 옵션, 예제를 표준 출력으로 보여주는 단순 함수입니다. 문자열 리터럴 안에 서식 플레이스홀더(`{program}`)가 있습니다.

## 280~339: 계약 파일 수집
- `280-311`: `discover_contracts`는 `.sol` 파일을 모아 읽고, 계약을 파싱하여 `ContractInfo`로 누적합니다.
- `313-339`: `collect_solidity_files`는 재귀적으로 디렉터리를 순회하며 `.sol` 확장자만 벡터에 추가합니다.

## 341~399: Solidity 계약 파싱
- `341-346`: 임시 구조 `ParsedContract` 정의.
- `347-399`: `parse_contracts_from_source`는 문자열에서 `contract` 키워드를 찾아 계약 이름과 본문을 추출합니다. `StringTracker`로 문자열 리터럴 내부를 건너뛰고, `is_identifier_boundary` 및 `is_abstract_contract`로 잘못된 매칭을 피합니다.

## 403~437: 생성자 파싱
- `403-437`: `parse_constructor_params`는 계약 본문에서 `constructor` 선언을 찾아 괄호 안 매개변수 목록을 반환합니다. 괄호 짝 맞춤은 `find_matching_paren`을 호출해 해결합니다.

## 439~472: 매개변수 분리
- `439-472`: `split_parameters`는 쉼표 단위로 매개변수를 분할하되, 튜플/배열 등 내부 괄호 깊이를 추적하는 `BracketTracker`를 활용합니다.

## 474~522: 매개변수 이름 추출
- `474-515`: 매개변수 문자열을 뒤에서부터 훑어 실제 이름 토큰을 찾습니다. `memory`, `indexed` 같은 예약된 접미사는 제거합니다.
- `517-522`: `is_reserved_suffix`로 예약어 목록을 정의합니다.

## 524~576: 주석 제거
- `524-576`: `strip_comments`는 `//`와 `/* */` 주석을 제거하되 문자열 리터럴과 줄바꿈을 유지합니다. `StringTracker`를 재사용합니다.

## 578~618: 계약 선택 로직
- `578-617`: `select_contract`는 사용자 입력을 정규화(슬래시 방향 통일)하고, 이름·파일 경로·파일명과 비교하여 일치 항목을 찾습니다.
- 중복이 있으면 경로와 함께 목록을 보여 주고 에러를 반환합니다.

## 620~628: JSON 인자 파싱
- `620-628`: `parse_args_json`은 `serde_json::from_str`로 파싱 후 배열인지 확인하고, 요소를 문자열로 되돌립니다.

## 630~668: 대화형 생성자 입력
- `630-668`: `prompt_for_args`는 매개변수 목록을 출력하고 한 항목씩 입력을 요구합니다. 빈 문자열이면 다시 반복합니다.

## 671~701: 프라이빗 키 처리
- `671-680`: 옵션으로 키가 들어오면 공백 여부만 확인하고 바로 반환합니다.
- `683-701`: `prompt_for_private_key`는 경고 문구를 출력하고, `rpassword::read_password`로 마스크된 입력을 받습니다.

## 704~779: 상대 경로 계산
- `704-719`: `compute_import_path`는 스크립트 파일과 계약 파일 사이의 상대 경로를 문자열로 변환합니다.
- `721-779`: `relative_path_between`은 서로 다른 프리픽스(Windows 드라이브 등)를 처리하고, 공통 경로를 제거한 뒤 `..`를 쌓아 상대 경로를 구성합니다.

## 781~812: 스크립트 렌더링
- `781-793`: 계약에서 가져온 `pragma`를 우선 사용하고 없으면 기본 버전 `^0.8.23`을 씁니다.
- `794-802`: 생성자 인자가 있으면 쉼표로 합쳐서 `new Contract(args...)` 구문에 삽입합니다.
- `804-812`: 완성된 Foundry 스크립트 문자열을 반환합니다. `vm.startBroadcast`, `vm.stopBroadcast`가 포함됩니다.

## 814~825: 계약 목록 출력
- `814-825`: `print_contract_list`는 `--list` 모드에서 계약 이름과 생성자 정보를 보기 좋게 출력합니다.

## 828~943: 파서용 헬퍼 함수들
- `828-842`: `find_next_char`는 특정 문자까지 전진하며, `{`를 찾을 때는 공백 외 문자도 허용합니다.
- `844-879`: `find_matching_brace/parens`는 문자열 상태를 고려해 괄호 짝을 찾습니다.
- `881-925`: 공백 건너뛰기와 식별자 감지에 필요한 유틸 함수들입니다.
- `927-952`: 식별자 앞뒤 문자를 확인하거나 `abstract` 키워드 여부를 판단합니다.

## 954~1058: 상태 추적기 구조체
- `964-1019`: `StringTracker`는 따옴표 내부 여부와 이스케이프 상황을 기록합니다.
- `1021-1058`: `BracketTracker`는 괄호/대괄호/중괄호 깊이를 추적해 쉼표 분기 시점을 알려 줍니다.

## 1061~1138: 단위 테스트
- 각 테스트는 파서 유틸 함수의 핵심 동작을 검증합니다.
  - `1066-1072`: 중첩 튜플 파라미터 분리 테스트.
  - `1075-1082`: 생성자 파라미터 추출 테스트.
  - `1084-1095`: 추상 계약을 무시하는지 확인.
  - `1097-1111`: 한 파일 내 여러 계약 처리.
  - `1113-1122`: 매개변수 이름 추출 테스트.
  - `1125-1136`: 주석 제거가 문자열을 보존하는지 확인합니다.

### 추가 학습 포인트
- **소유권/빌림**: `PathBuf`와 문자열을 복사(`clone`)하는 위치를 살펴보세요 (`280-307`, `620-628`).
- **`?` 연산자**: 에러 전파 흐름을 따라가며 어디서 조기에 반환되는지 추적해 보세요 (`18-125`).
- **이터레이터 사용법**: `map`, `filter`, `collect`, 수동 반복 등을 찾아보세요 (`160-166`, `584-597`, `439-472`).
- **문자열 처리**: `trim`, 슬라이싱, `to_string` 변환 등 러스트 문자열 API 사용 예시가 풍부합니다 (`474-515`).

> 노트: 실제 Solidity 문법이 더 복잡해지면 외부 파서를 사용하는 것이 안전합니다. 본 코드는 러스트 문자열 파싱 연습으로 이해하면 좋습니다.
